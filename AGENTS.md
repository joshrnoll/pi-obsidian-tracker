# Repository Guidelines

## Purpose

This repository contains Pi skills for an Obsidian-vault issue tracker workflow.

## Conventions

- Keep skills small, explicit, and composable.
- Treat `templates/` as the canonical file contract.
- Treat `docs/` as explanatory material, not executable state.
- Keep board markdown compatible with the Obsidian Kanban plugin.
- Prefer conservative file edits over clever transformations.
